# 二元期权价格行为交易系统（BOPA System v1.0）

> Binary-Option Price-Action System
> 适用标的：BTCUSDT（及其他高流动性合约）
> 触发周期：1m 滚动；可交易期限：5M / 10M / 15M / 30M
> 数据底座：`BINARY_OPTIONS_FEATURES_V3`（feature_pool / 相位 / 三层打分 / 关键事件）
> 执行主体：四 Agent（历史·分析·批评·裁判）协作 + 本规则引擎

---

## 0. 这套系统为什么和「股票/合约」不一样（设计第一性原理）

二元期权的本质决定了交易系统的全部取舍：

| 特性 | 后果 | 系统设计取向 |
|---|---|---|
| **固定周期判涨跌** | 只赌「到期时收盘价 vs 入场价」的方向，与涨幅无关 | 只需预测**方向 + 方向能维持到到期**，不预测目标价 |
| **中途无法干预** | 没有止损、没有移动止盈、没有加减仓 | **入场即定生死**，入场质量 = 系统全部价值 |
| **赔率固定（约 70%~90%）** | 盈亏比锁死，无法用大盈亏比覆盖低胜率 | **胜率为王**：盈亏平衡胜率约 53%~59%，系统必须把命中率推到 60%+ |
| **到期时间是变量** | 同一方向，5M 对、30M 可能错 | **期限 = 对“这段方向能走多久”的时间投影**，是独立决策维度 |

> **一句话纲领**：在「高时间框架趋势方向」上，等「低时间框架回撤结束/突破确认」的那一根**信号K线**，用**与预期推进时长匹配的期限**入场；用**量价**确认真伪，用**做市商陷阱过滤器**剔除假动作，用**相位**一票否决力竭追单。

### 0.1 胜率门槛（必须背下来）

赔率 `p`（净收益率）对应的盈亏平衡胜率 `W* = 1 / (1 + p)`：

| 平台赔率 p | 盈亏平衡胜率 W* | 系统目标胜率 |
|---|---|---|
| 0.70 | 58.8% | ≥ 64% |
| 0.80 | 55.6% | ≥ 61% |
| 0.85 | 54.1% | ≥ 60% |
| 0.90 | 52.6% | ≥ 59% |

**含义**：任何一笔交易，如果估计命中率 < 对应 `W* + 5%` 的安全垫，就应当**观望（NO-TRADE）**。宁可不做，不做亏期望的单。

### 1.1 《价格行为交易系统详解》（Al Brooks 体系：趋势/通道/区间 + 信号K线）

| 核心理念 | 落地规则 | 对应数据字段 |
|---|---|---|
| **Always-In（永远在场方向）**：任意时刻市场只有一个“当前主导方向” | 用 15M+30M 的 EMA 排列与结构定主导方向 | `15m/30m.trendBias`、`ema9/21/55`、`swing.structure`、`layered_score.phase.trendDir` |
| **趋势 vs 区间 vs 通道** | ADX/BBW 判环境，结构判走法 | `adx`、`bbw`、`marketState`、`swing.highSeq/lowSeq` |
| **回撤进场（H1/H2、L1/L2）**：趋势中第一次/第二次回撤后顺势 | 回踩 EMA21 不破结构 + 反向拒绝针 | `triggerFlags:["reclaim_ema21"]`、`dist.ema21_atr`、`pinBar` |
| **突破 + 回踩确认（Breakout & Retest）** | 破前高后回踩不破，再顺势 | `swing.brokePrevHigh/brokePrevLow`、`prevHigh/prevLow`、`dist.prevHigh_atr` |
| **测量移动（Measured Move）** | 用 ATR 估算「这段还能走多远/多久」→ 选期限 | `atr`、`dist.*_atr` |
| **信号K线质量（trend bar / doji）** | 实体占比、收盘位置定信号强弱 | `bodyRatio`、`closeLocation`、`upperWickRatio`、`lowerWickRatio` |
| **微通道 / 过度伸展（climax）** | 连续同向且远离均线 = 力竭风险 | `dist.ema21_atr`（>2 伸展）、`phase.phase="exhaustion"` |

### 1.2 《量价分析精要》（Anna Coulling / VPA：量是因，价是果）

| 核心理念 | 落地规则 | 对应数据字段 |
|---|---|---|
| **量价配合 vs 背离**：上涨要放量，缩量上涨可疑 | 放量突破才信，缩量突破降级 | `volRatio`、`obvSlope`、`momDivergence` |
| **高潮量（Climax / 抛压/吸筹高潮）** | 极端放量 + 小实体 = 多空换手，方向可能反转 | `volRatio`（>2.5~3）+ `bodyRatio`（小） |
| **无需求/无供给（No Demand / No Supply）** | 缩量小阳/小阴 = 该方向无人接力 | `volRatio < 0.7` + 小实体 + `volRegime="contracting"` |
| **努力 vs 结果（Effort vs Result）** | 大量却走不动 = 有反向力量吸收 | `volRatio` 高但 `dist.*_atr` 几乎不变 / `bodyRatio` 小 |
| **OBV/累积量验证趋势** | OBV 与价同向才健康；背离预警 | `obvSlope`、`momDivergence:"bull"/"bear"` |
| **成交量区制（扩张/收缩）** | 收缩末端常有突破，扩张中段是行情主体 | `volRegime`、`volRegimeRatio` |

### 1.3 《Forex Price Action Scalping》（Bob Volman：极简裸 K 短线）

| 核心理念 | 落地规则 | 对应数据字段 |
|---|---|---|
| **20EMA 作为动态趋势线/价值线** | 价格相对 EMA 的位置=偏向；回踩 EMA 是顺势机会 | `ema9/ema21`、`dist.ema21_atr` |
| **整数关口（round number）磁吸与阻断** | 临近整数位要警惕假突破/反转 | `roundLevel`、`dist.roundLevel_atr` |
| **假突破（False Break / Tease Break）** | 刺破前高/前低后迅速收回 = 陷阱，反向机会 | `pinBar`、`upperWickRatio/lowerWickRatio`、`brokePrevHigh/Low` + 收盘回内 |
| **筑顶/筑底（Building Highs/Lows）** | 连续同向小K堆叠后突破更可信 | `barsDirection3/5`、`swing.highSeq/lowSeq` |
| **回撤反转入场（Pullback Reversal）** | 回撤到 EMA/前破位后出现反向信号K | `triggerFlags`、`stochTurn`、`stochK/stochD` |
| **天花板/地板（Ceiling/Floor）= 区间边界** | 区间内在边界做反向，区间中部观望 | `bbPct`（0~1 在布林带内位置）、`prevHigh/prevLow` |

### 1.4 《做市商优势》（Josh Lukeman / Market Maker 视角：反向思考散户陷阱）

| 核心理念 | 落地规则 | 对应数据字段 |
|---|---|---|
| **止损猎杀（Stop Hunt / 流动性扫荡）** | 价格刺破明显高低点（散户止损区）后反向 = 庄家取量 | `brokePrevHigh/Low` + 长上下影 `pinBar` + 收盘回内 |
| **诱多/诱空（Trap）** | 最“显而易见”的突破往往是陷阱 | `volRatio` 异常 + `bodyRatio` 小 + 临近 `roundLevel` |
| **吸筹/派发区** | 区间内大量换手而价不动 = 在建仓/出货 | `marketState="震荡"` + 高 `volRatio` + 横向结构 |
| **逆向于拥挤交易** | 极端超买超卖 + 一致性信号 = 反向风险升高 | `rsi` 极值、`stochK/D` 极值、`bbPct>1 或 <0` |
| **真突破的足迹** | 突破要有量、有跟随、不立即回吞 | `volRatio`↑ + `barsDirection` 跟随 + 不触发反向 `pinBar` |

### 1.5 《短线交易真好玩》（短线动能/波动率突破 + 纪律）

| 核心理念 | 落地规则 | 对应数据字段 |
|---|---|---|
| **波动率压缩→扩张（挤压后爆发）** | BBW 极窄是“蓄势”，爆发方向跟随首根放量突破 | `bbw`、`volRegime="contracting"`、布林带挤压事件 |
| **顺动能、不接飞刀** | 动能（MACD 柱/RSI 斜率）转向前不抢反转 | `macdHist` 序列、`rsi` 近12根序列、`stochTurn` |
| **日内时段与节奏** | 用相位区分「推进/回撤/力竭」，只在推进与回撤末做单 | `layered_score.phase` |
| **严守纪律、限制连亏** | 连亏熔断、每日上限、固定单注 | 见 §12 资金纪律 |
| **简单可重复 > 复杂** | 规则引擎化，减少临场情绪 | 本文件 §3~§9 规则即引擎 |

### 1.6 五本书 → 系统三大支柱

```
支柱A：方向（顺势）   = 价格行为(Brooks) + Volman(EMA/结构)  → 三层打分的“趋势分”
支柱B：真伪（确认）   = 量价分析(VPA)   + 做市商优势(反陷阱)  → “触发分/否决项”
支柱C：时机与时长     = 短线动能(挤压/动能) + 相位            → “期限选择 + 相位一票否决”
```

## 3. STEP 0：相位闸门（系统的“安全开关”）

相位是**第一权重**，凌驾于任何高总分之上。直接读 `layered_score.phase`。

| `phase.phase` | 含义 | 允许动作 | 期限要求 |
|---|---|---|---|
| `impulse`（推进段） | 顺势推进、未过度伸展 | 强顺势（与 `trendDir` 同向） | 短期限优先（5M/10M） |
| `pullback`（回撤段） | 顺势中的逆向回踩 | 可顺势，但**只在回撤结束信号出现后** | **强制长期限 15M/30M** |
| `exhaustion`（力竭段） | 趋势末端，动能衰竭 | **禁止顺势追单**；只可观望或反向（需极强证据） | — |
| `reversal`（反转段） | 结构破坏，方向切换中 | 观望，待新方向确认 | — |

**铁律**：
- `phase.suppress = true` → 不论 `total` 多高，**该延续方向一律观望**。这是为了防止「在力竭顶/底追单」和「在回撤里反复被打脸」。
- `retraceType = "pullback"` → 顺势单**必须**用 `phase.suggestExpiry`（长期限），不许用 5M 抢。
- 力竭判定的「衰竭三件套」必须成立才认（交给 Critic 复核）：**动能背离 + 缩量 + StochRSI 反向转**。三缺二则降级为「中性推进」，不直接给反向。

## 4. STEP 1：市场环境判定（决定用哪套剧本）

环境决定**剧本选择**：趋势环境用「顺势回撤/突破」，区间环境用「边界反转」，转换/挤压环境用「等突破首根」。

### 4.1 环境判定表（以 env_tf = 15M / 30M 为主）

| 维度 | 趋势环境 | 区间/震荡环境 | 挤压（蓄势） |
|---|---|---|---|
| `adx` | ≥ 25 | < 20 | 任意（看 BBW） |
| `bbw`（带宽%） | 正常~扩张 | 中性 | **处于近 50 根最低 10%** |
| `volRegime` | flat / expanding | flat | **contracting** |
| `swing.structure` | up / down | range | range |
| `marketState` | 趋势 / 趋势形成 | 震荡 | 震荡 |

### 4.2 剧本路由

```
IF 环境=趋势：
    剧本 = 顺势剧本（§5 方向规则中的“趋势分”主导）
    入场 = 回撤结束 或 突破回踩确认
ELIF 环境=区间：
    剧本 = 区间反转剧本
    入场 = 价在区间边界(bbPct≥0.8 顶 / ≤0.2 底)出现反向拒绝针，做向中轨方向
    区间中部(0.35<bbPct<0.65) → 观望
ELIF 环境=挤压：
    剧本 = 突破跟随
    动作 = 不预测方向；等首根“放量+大实体+收在极值”的突破K，顺其方向做
    禁止 = 在挤压内部对赌方向（这是亏损重灾区）
```

> **关键事件「布林带挤压 / 喇叭口张开」**直接驱动「挤压剧本」。挤压未破时 = 观望；喇叭口张开 + 放量 = 突破已启动，顺势。

## 5. STEP 2：三层打分定方向（对接 `layered_score`）

直接复用你已有的三层结构，但把每层的**判据标准化**，让四个 Agent 与规则引擎口径一致。

### 5.1 第一层：1H 方向偏向（bias，权重最高的“底色”）

| 条件 | 评分 |
|---|---|
| `1h.ema21 > ema55` 且 `price > ema21` | 偏多 +2 |
| `1h.ema21 > ema55` 但 `price < ema21` | 偏多 +1 |
| EMA 交叉/混乱（如本例 ema55>ema21） | 中性 0 |
| `1h.ema21 < ema55` 且 `price < ema21` | 偏空 -2 |

> 1H 中性（0）时：**不允许重仓任何方向**，只能做「低时间框架高质量回撤单」且降一档置信度。

### 5.2 第二层：15M 信号分（环境层，0~±5）

| 信号 | 多头计分 | 字段 |
|---|---|---|
| EMA 多头/空头排列 | ±1 | `15m.ema9/21/55` |
| RSI 动能方向（>50 多 / <50 空，且斜率同向） | ±1 | `15m.rsi` + 近12根序列斜率 |
| 价在 VWAP 上/下方 | ±1 | `dist.vwap_atr` 符号 |
| 结构 HH/HL（多）或 LH/LL（空） | ±1 | `swing.structure` |
| MACD 柱方向与扩张 | ±1 | `15m.macdHist` 近8根序列 |
| **否决：BBW 极窄（横盘无波动）** | 触发硬否决标记 | `bbw` 在最低 10% |

### 5.3 第三层：5M / 1M 触发分（执行层，0~±5）

| 触发 | 计分 | 字段 |
|---|---|---|
| 回踩 EMA21 支撑/阻力不破 | ±1 | `triggerFlags:["reclaim_ema21"]`、`dist.ema21_atr` |
| 反向拒绝针（pin / 长下影/上影） | ±1 | `pinBar`、`lowerWickRatio/upperWickRatio` |
| 放量突破前高/前低（真突破） | ±2 | `volRatio>2` + `brokePrevHigh/Low` + 大实体 |
| StochRSI 自极值反转（金叉/死叉） | ±1 | `stochTurn`、`stochK/stochD` |
| 吞没形态顺势 | ±1 | `engulfing` |

### 5.4 方向合成（Direction Engine）

```
DIR_SCORE = bias1h(×权重1.0) + signal15m(×权重0.8) + trigger(×权重0.6) + phase调整
方向：
   DIR_SCORE ≥ +4  → 看涨（CALL）候选
   DIR_SCORE ≤ -4  → 看跌（PUT）候选
   -4 < DIR_SCORE < +4 → 观望（方向不清）
置信度（用于期限与仓位）：
   |DIR_SCORE| ≥ 7  → 高置信
   5 ~ 6           → 中置信
   4               → 低置信（仅长期限 + 最小注 或 观望）
```

> 这一步**只产生“候选方向”**。候选必须通过 §6 量价确认 + §7 反陷阱 + §8 硬否决，才能成为正式信号。

## 6. STEP 3a：量价确认模块（VPA — 去伪存真的第一关）

候选方向必须接受量价检验。**价是结果，量是原因**；没有量的配合，方向不可信。

### 6.1 量价确认矩阵

| 价格行为 | 健康（确认，信号 +1 档） | 可疑（降级或否决） |
|---|---|---|
| 向上突破/上涨 | `volRatio ≥ 1.5` 且 `obvSlope > 0` 且大实体 | 缩量上涨 `volRatio < 0.8` 或 `obvSlope < 0`（**背离**） |
| 向下突破/下跌 | `volRatio ≥ 1.5` 且 `obvSlope < 0` 且大实体 | 缩量下跌 / OBV 不创新低 |
| 回撤（顺势中的逆向） | **缩量回撤** `volRatio < 1`（健康，说明逆势无力） | 放量回撤（危险，可能是真反转，不是回撤） |
| 突破后 | 不立即被反向吞回 | 放量后小实体（高潮量）→ 换手/吸收 |

### 6.2 量价红旗（出现即降一档置信度，或交 Critic 复核）

- **高潮量陷阱**：`volRatio ≥ 2.5` 且 `bodyRatio ≤ 0.3`（巨量小实体）→ 多空激烈换手，方向易反转，**禁止顺该K方向追**。
- **OBV 背离**：`momDivergence` 非空，或某 TF `obvSlope` 与价格方向相反 → 趋势内部走弱预警。
- **努力无果**：放量（`volRatio` 高）但 `dist.*_atr` 几乎无变化（价格走不动）→ 有反向力量吸收。
- **无需求/无供给**：`volRatio < 0.7` + 小实体 + `volRegime="contracting"` → 该方向无接力，突破不可信。

### 6.3 量价确认产出

```
VPA_STATE ∈ { CONFIRM(+1档), NEUTRAL(0), DIVERGE(-1档或否决) }
规则：
  候选方向与 VPA_STATE 同向且 CONFIRM → 升档
  DIVERGE 且为“放量反向/高潮量” → 该方向降为低置信 或 直接 NO-TRADE
```

## 7. STEP 3b：做市商反陷阱过滤器（Anti-Manipulation Filter）

> 核心立场（《做市商优势》）：**最显而易见的突破，往往是为收割散户止损而设的陷阱。** 在二元期权里被一根假突破打到到期，就是直接判负，因此反陷阱是高优先过滤。

### 7.1 陷阱识别清单（命中 = 反向警惕 / 否决顺势追单）

| 陷阱类型 | 识别特征（字段） | 处理 |
|---|---|---|
| **止损猎杀 / 流动性扫荡** | `brokePrevHigh/Low = true` 但收盘**回到**前高/前低内侧 + 长上/下影 `pinBar` | **反向**信号（扫上影→看跌；扫下影→看涨），优先级高 |
| **整数关口假突破** | 价格触及 `roundLevel`（`\|dist.roundLevel_atr\| < 0.5`）后被拒 | 禁止在整数位顺势追突破；倾向反向 |
| **诱多/诱空巨量陷阱** | `volRatio` 异常高 + 小实体 + 临近关键位 | 视为换手，不追 |
| **区间内假突破** | `marketState=震荡` 中刺破区间边界后快速收回 | 反向做回区间，期限取短 |
| **一致性极端（拥挤交易）** | RSI 极值 + StochRSI 极值（`>90` 或 `<10`）+ 价在布林带外（`bbPct>1 或 <0`） | 顺势降级；反向需衰竭三件套 |

### 7.2 真突破 vs 假突破判别（Volman 的“足迹”）

```
真突破（可顺势）需同时满足：
  1) 放量：volRatio ≥ 1.5（突破K或其前一两根）
  2) 实体：bodyRatio ≥ 0.5 且 closeLocation 收在突破方向极值（>0.7 或 <0.3）
  3) 跟随：barsDirection3/5 与突破同向，不立即出现反向 pinBar
  4) 位置：突破点不紧贴 roundLevel 的“吸磁/阻断”侧
假突破（反向或观望）：
  上述任一关键项缺失，尤其“刺破后收盘回内 + 长影线”→ 判为陷阱
```

## 8. STEP 4：硬否决清单（No-Trade Filter，命中即观望/降级）

> 二元期权没有止损，**“不交易”本身就是最强的风控**。命中下列任一**红色否决项**：直接 `NO-TRADE`。命中**黄色降级项**：置信度降一档并强制最小注。

### 8.1 红色否决（一票 NO-TRADE）

1. `phase.suppress = true`（力竭/反转相位）且欲做顺势单。
2. 环境=**挤压且未突破**（`bbw` 最低 10% + `volRegime=contracting`），却要在挤压内对赌方向。
3. 方向分 `|DIR_SCORE| < 4`（方向不清）。
4. 高潮量陷阱（`volRatio ≥ 2.5` + `bodyRatio ≤ 0.3`）出现在欲追方向的触发K上。
5. 多周期**剧烈矛盾**：1H/30M/15M 中有两个给出相反的强信号（如 30M 多 vs 1H 强空），且无相位裁决。
6. 重大数据/事件窗口前后（外部输入，若有）。

### 8.2 黄色降级（降一档 + 最小注 + 仅长期限）

1. 1H 方向中性（bias=0）。
2. 存在 OBV/动能背离（`momDivergence` 非空）。
3. 触发K紧贴 `roundLevel`（`|dist.roundLevel_atr| < 0.5`）。
4. `volRatio < 0.8`（量能不足，突破/推进可信度低）。
5. 价格过度伸展（`|dist.ema21_atr| > 2`）——追单到期回归风险高。

### 8.3 与现有「no_trade_reasons」对接

`expiry_candidates[].no_trade_reasons` 已经输出如「波动率压缩过窄」。本系统将其映射为：
- 「波动率压缩过窄」→ 红 #2（若未突破）/ 黄 #4（若已放量突破）。Critic 必须判定“突破是否已发生”。

## 9. 价格行为信号K线形态库（执行层识别）

入场永远落在**一根具体的信号K线**上。下表是允许触发入场的形态及其字段判据。

| 形态 | 多头判据 | 空头判据 | 主要字段 |
|---|---|---|---|
| **顺势趋势K（trend bar）** | 大阳实体收高 `bodyRatio≥0.6, closeLocation≥0.7` | 大阴实体收低 `closeLocation≤0.3` | `bodyRatio/closeLocation` |
| **回撤拒绝针（pin bar）** | 长下影、收在上半部、在 EMA21/支撑处 | 长上影、收在下半部、在阻力处 | `pinBar/pinBarValid/lowerWickRatio/upperWickRatio` |
| **吞没（engulfing）** | 阳吞前阴，顺多 | 阴吞前阳，顺空 | `engulfing` |
| **突破回踩确认** | 破前高→回踩不破前高→再起 | 破前低→回踩不破→再落 | `brokePrevHigh/Low`+`dist.prevHigh/Low_atr` |
| **假突破反转（陷阱）** | 刺破前低收回 + 长下影 | 刺破前高收回 + 长上影 | §7.1 止损猎杀 |
| **区间边界反转** | `bbPct≤0.2` + 下影拒绝 | `bbPct≥0.8` + 上影拒绝 | `bbPct`+`pinBar` |
| **StochRSI 反转** | 自 `<20` 金叉上行 | 自 `>80` 死叉下行 | `stochTurn/stochK/stochD` |

> **信号K质量打分**：实体占比、收盘位置、是否在关键位、是否有量。质量越高，置信档越高。`pinBarValid=false` 的 pin 不计为有效触发。

## 10. STEP 5：期限选择（把方向投影到时间）

期限是**独立维度**：方向对、期限错，照样判负。原则——**期限 ≤ 当前推进/动能可维持的时长**。

### 10.1 期限选择决策表

| 情形 | 推荐期限 | 依据 |
|---|---|---|
| `phase=impulse` 推进段 + 高置信 + 放量突破 | **5M / 10M** | 推进段动能集中在近端，短打命中率高 |
| `phase=pullback` 回撤后顺势 | **15M / 30M**（强制） | 回撤需要时间消化，短期限易被噪声打穿 |
| 区间边界反转（做回中轨） | **5M / 10M** | 反转回归通常快速 |
| 挤压突破首根跟随 | **10M / 15M** | 突破初段需要时间展开 |
| 低置信（DIR_SCORE=4）/ 1H 中性 | **15M / 30M** 或观望 | 长期限平滑噪声，降低单根 K 误判 |
| 高 ATR/扩张期（波动大） | 相对缩短一档 | 波动大时方向更快兑现，长持反被反向波及 |

### 10.2 与 `expiry_candidates` 对接

系统**不盲从** `best_candidate`。流程：
1. 读 `expiry_candidates` 的 `total_score` 作为**初选排序**。
2. 用 §10.1 的**相位/剧本约束**覆盖：相位 = pullback 时即使 5M 总分最高，也**禁用 5M**，下移到 15M。
3. 扫描 `no_trade_reasons`，命中红色否决则该期限作废。
4. 剩余候选中，选「相位允许 + 置信匹配 + 无红旗」里**总分最高**者。

### 10.3 期限与置信度的联动

```
高置信 + impulse        → 5M（激进）/10M（稳健）
中置信                  → 10M / 15M
低置信（但可做）        → 仅 15M / 30M，最小注
任何“被迫做短”的冲动    → 默认拒绝，长期限更安全
```

## 11. 四 Agent 协作工作流（Historian / Analyst / Critic / Judge）

系统的执行不是单点判断，而是**四角色串行 + 裁判收口**，对接 `agent_views` 的分工。每个 Agent 有**唯一职责、固定输入、结构化输出**，避免越权与重复。

### 11.1 角色定义与职责边界

```
①历史 Historian —— “这种局面以前怎么走？”
   输入：layered_score.phase、trendBias、volRegime、momDivergence、dist.*_atr
   任务：仅检索“相位相同”的历史样本（相位不同则历史不可比，直接声明不可比）
   输出：HistoricalPrior{ comparable:bool, sampleBias, winrateHint, note }
   禁止：给最终方向（它只提供先验概率）

②分析 Analyst —— “综合给出方向与置信”
   输入：全 feature_pool + 三层打分 + Historian 先验
   判断顺序（强制）：
     1) 先读 phase：suppress=true → 该方向观望；pullback → 顺势但长期限
     2) 再用 DIR_SCORE + feature_pool 定方向与置信
   输出：AnalystView{ direction, confidence, expiry, scriptUsed, reasons[] }

③批评 Critic —— “挑毛病，证伪”
   输入：AnalystView + 原始数据
   核查清单（强制逐条）：
     - 相位证据是否成立？（力竭三件套 / 回撤“结构未破+缩量”是否属实）
     - 是否放量破结构（=反转）被误判成回撤？
     - 量价是否背离？是否高潮量陷阱？
     - 是否撞 roundLevel / 假突破 / 止损猎杀？
     - 硬否决清单是否有漏网？
   输出：CritiqueReport{ vetoes[], downgrades[], confirmations[], adjustedConfidence }

④裁判 Judge —— “拍板，相位第一权重”
   输入：AnalystView + CritiqueReport + HistoricalPrior
   规则：
     - phase.suppress=true → 不论总分多高，倾向 NO-TRADE
     - retraceType=pullback → 顺势单必须长期限(15/30M)
     - Critic 命中任一红色否决 → NO-TRADE
     - 否则按 (置信度×历史先验) 给最终决策与期限
   输出：FinalDecision{ action, expiry, confidence, size, rationale }
```

### 11.2 协作时序

```
feature_pool 生成（每 1m）
        │
        ▼
 [Historian] ──先验──► [Analyst] ──观点──► [Critic] ──证伪──► [Judge] ──决策──► 执行/记录
        ▲                                                        │
        └──────────────── 结果回灌（用于历史样本库）◄────────────┘
```

### 11.3 决策仲裁优先级（冲突时谁说了算）

```
1. 相位闸门（phase.suppress / pullback 期限约束）       —— 最高，谁都不能推翻
2. Critic 的红色否决（硬否决清单 §8.1）                —— 直接 NO-TRADE
3. Judge 综合（方向×置信×历史先验）                    —— 在 1、2 通过后定档
4. Analyst 方向                                        —— 提供候选
5. Historian 先验                                      —— 仅调整置信，不定方向
```


**系统最终裁决：低置信局面。首选【观望】；如规则要求必须出方向，则【15M 看涨 · 最小注（0.5档）】，明确禁止 5M 追多与逼近 74000 的重仓。**

> 这个例子的价值在于：原始 `best_candidate` 给的是「5M、总分13、看涨」，而本系统通过**量价背离 + 整数关口 + 1H 超买挤压 + 动能衰减**把它降级为「观望/最小注 15M」。**这正是五本书共同强调的：高分诱多处，往往是做市商的收割区。**

## 13. 资金与纪律管理（二元期权专用）

二元期权无止损，资金管理是**唯一的生存机制**。

### 13.1 单注规模（固定分注 + 置信分档）

| 置信度 | 单注（占账户总额%） | 允许期限 |
|---|---|---|
| 高 | 2%（基准 1 档 = 2%） | 5M/10M |
| 中 | 1%（0.5 档） | 10M/15M |
| 低 | 0.5%（最小注）或观望 | 15M/30M |

> 永远**固定百分比**，禁止马丁加倍（连错时马丁会瞬间爆仓）。单注上限硬封顶 2%。

### 13.2 熔断规则（强制停手）

```
连续亏损 3 笔        → 暂停当前时段，复盘后再开
单日亏损达账户 6%    → 当日停止交易
单日已盈利 8%        → 落袋，降频或收工（防回吐）
胜率滚动20笔 < W*    → 暂停该剧本，重检参数
相位=力竭/反转 整段  → 该方向当段不做
```

### 13.3 交易日志字段（用于回灌 Historian 与迭代）

```
ts | symbol | phase | script | direction | expiry | dir_score |
vpa_state | vetoes/downgrades | confidence | size | result(win/lose) | pnl |
入场时关键位距离(dist.*_atr) | 备注(事后复盘:错在方向还是期限?)
```

> **区分“方向错”还是“期限错”**：若到期前方向正确、到期瞬间被打回 = 期限选短了；若全程逆向 = 方向判错。两类错误的修法完全不同。

## 14. 参数速查表（Cheat Sheet）

### 14.1 指标阈值

| 指标 | 趋势/强 | 中性 | 弱/反向 |
|---|---|---|---|
| ADX | ≥25 趋势 | 20~25 转换 | <20 震荡（趋势策略失效） |
| RSI | >55 偏多 / <45 偏空 | 45~55 | 极值 >70 / <30 警惕 |
| StochRSI | 20~80 区间内顺势 | — | >80超买 / <20超卖（反转关注） |
| volRatio | ≥1.5 放量确认；≥2.5 高潮警惕 | 0.8~1.5 | <0.8 量不足/无需求 |
| BBW | 扩张=行情 | 正常 | 最低10%=挤压（突破前观望） |
| bbPct | — | 0.35~0.65 中部观望 | ≥0.8 顶 / ≤0.2 底（边界反转） |
| dist.ema21_atr | <1 健康回踩 | 1~2 | >2 过度伸展（不追） |
| dist.roundLevel_atr | — | — | \|·\|<0.5 整数关口阻断区 |

### 14.2 一页决策口诀

```
1. 先看相位：力竭/反转 → 不做；回撤 → 长期限；推进 → 可短打
2. 再看环境：趋势顺势 / 区间做边界 / 挤压等突破首根
3. 三层定向：1H底色 → 15M信号 → 5M触发，≥+4才算方向清
4. 量价证伪：放量配合才信；OBV背离/高潮量/缩量突破 → 降级
5. 反陷阱  ：刺破收回=陷阱反向；逼近整数位不追；显而易见的突破要怀疑
6. 扫否决  ：红旗任一→观望；黄旗叠加→最小注+长期限
7. 选期限  ：期限≤动能可维持时长；拿不准就拉长、就观望
8. 守纪律  ：固定分注、连亏熔断、不马丁、记日志分清“方向错/期限错”
```

### 14.3 信号强弱总览（最终 action 映射）

| 综合状态 | action | 期限 | 仓位 |
|---|---|---|---|
| 相位推进 + 方向≥7 + 量价确认 + 无红旗 | CALL/PUT | 5M/10M | 2% |
| 方向5~6 + 量价中性 + ≤1黄旗 | CALL/PUT | 10M/15M | 1% |
| 方向=4 或 1黄~2黄 + 背离 | CALL/PUT（弱） | 15M/30M | 0.5% |
| 红旗任一 / 相位suppress / 方向<4 / 挤压未破 | **NO-TRADE** | — | 0 |

## 15. 落地与接入说明

### 15.1 这套系统怎么用
1. 每根 1m K 收盘后，数据层产出 `BINARY_OPTIONS_FEATURES_V3`。
2. 按 §2 漏斗依次执行 STEP0→STEP5，四 Agent（§11）串行收口。
3. Judge 输出 §11.4 的 `FinalDecision` JSON，由执行层在「入场时刻」下单。
4. 到期结果回灌交易日志（§13.3）与 Historian 样本库。

### 15.2 与现有字段的对接锚点（无需改数据层）
- 相位 → `layered_score.phase`
- 三层分 → `layered_score.bias1h / signal15m / trigger5m`
- 期限候选 → `expiry_candidates` + `best_candidate`（本系统只作初选，受相位/否决覆盖）
- 否决 → `no_trade_reasons` + `vetoes`
- 关键位/距离 → `key_levels` + `feature_pool[tf].dist.*_atr`
- Agent 分工 → `agent_views.{historian,analyst,critic,judge}_focus`

### 15.3 迭代方向（建议）
- 按「相位 × 剧本」分桶统计胜率，剔除长期低于 W* 的桶。
- 校准各 TF 权重与 phase 加分（用真实回测胜率反推，而非拍脑袋）。
- 为 `roundLevel`、止损猎杀等陷阱单独统计命中率，验证反陷阱过滤器的有效性。
