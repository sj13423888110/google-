# v3.13.1 相位引擎重写（依五本书核心：趋势/回踩/反转 = 斜率结构+动能+量能 共同决定）

## 修复的核心问题
上涨趋势中分析师/裁判一路看涨、回踩全部亏损。根因：方向被死锁在 15M/30M 高周期，执行周期(5M)反向后无人能否决。

## 改了什么（content.js `_phaseEngine` 全量重写）
- **方向不再死锁高周期**：高周期(15M+30M)=背景偏向(bgDir，顺风/逆风)；执行周期(5M)用 **斜率结构 S + 动能 M + 量能 V 加权投票**(scores.op) 解析"当前可交易方向 tradeDir"。
- **不搞硬共振**(采纳你的意见，避免错过机会)：三维加权打分竞争，谁强归谁。
- **新相位语义**：
  - `retrace_progress` 回踩进行中 → **观望**(等回踩结束，严禁途中接刀，这是回踩亏损的根治)
  - `retrace_end` 回踩结束 → 顺势(短周期可入)
  - `reversal_confirmed` 反转确认 → **顺执行周期方向(可反手)**，依据=结构破位/放量+合力强(有迹可循，非乱反手)
  - `impulse` 推进 / `exhaustion` 力竭(≥2项衰竭) / `unclear` 方向不清
- **方向以 phase.tradeDir 为准**(不是高周期 bgDir)；4 个 Agent 提示词同步：方向取 tradeDir，回踩进行中观望，反转确认跟随。

## 离线验证（构造你截图场景）
- 截图(上涨背景+5M回踩) → 观望 ✅(不再接刀)
- 5M放量破位 → 反手看跌 ✅
- 回踩结束/顺势推进 → 看涨 ✅

# v3.13.2 纯剥头皮模式（Volman：执行周期定方向，大周期不锁死）

## 为什么
剥头皮超短线(1-5M)用 30M/1H 锁方向，书里(Volman/短线交易真好玩/Brooks)是反对的：在哪个周期交易就以哪个周期为 always-in 方向，大周期只作背景。上一版仍让大周期对立时"回踩观望"，对超短线偏慢偏锁。

## 改了什么（content.js `_phaseEngine`）
- **方向 = 执行周期(1M/5M)合力 opDir**：|op|≥2.5 直接定向，顺逆大周期都给。
- **大周期(15M/30M)降级为顺风/逆风**，只调期限与置信，绝不否决：
  - 顺风(tailwind)：impulse，可短打
  - 逆风(headwind)：counter_scalp / reversal_confirmed —— 方向照给，缩 5M 快进快出、可降注
  - 大周期中性：纯看执行周期
- **力竭**：只在 |op|≥4 且伸展到极值(2.0ATR/bbPct≥0.9) + ≥2 衰竭时才短暂观望（门槛高，少误杀）。
- 删除"回踩进行中→观望"逻辑（那是趋势交易思路，不适合剥头皮）。
- 4 Agent 提示词同步：方向取 phase.tradeDir，期限默认 5M，逆大周期照做，严禁因大周期方向否决执行周期信号。

## 离线验证
- 截图场景(大周期涨+5M跌) → **看跌 5M** ✅(不再被锁观望)
- 放量破位 → 看跌 5M(反转确认) ✅
- 顺势上涨 → 看涨 5M ✅

# v3.13.3 动态到期引擎（到期周期与方向同等重要，由书的核心算出，不锁死）

## 为什么
到期周期(赌多久)和方向同等重要，不能默认锁 5M。书的核心(Brooks 测量移动 measured move / Volman 动能续航 / 短线交易波动率)讲的就是：到期时长 = 这段方向"能维持多久/走多远"的时间投影。

## 改了什么（content.js 新增 `_expiryEngine`）
到期周期由 4 个维度动态算出 5/10/15/30：
1. **动能续航**：|op| 合力越强 → 趋势能延续 → 可长
2. **波动率(兑现速度)**：ATR/价格大 → 快速兑现 → 宜短；bbw 极窄(挤压) → 突破后持续 → 略长
3. **到关键位距离(measured move 目标)**：顺势方向前方阻挡近 → 短；空间大(≥3ATR) → 长
4. **入场类型**：反转/逆风 → 快进快出 → 短；顺风推进 → 给时间展开 → 可长；已伸展 → 短

- 删除所有写死的"默认 5M / 倾向 5M"。
- 输出带 `expiryWhy`(到期理由)，面板与提示词都显示依据。
- 4 Agent 提示词同步：期限以 phase.suggestExpiry 为准，可微调一档但须说明依据，严禁无脑锁 5M。

## 离线验证（同样看涨，周期不同）
- 强顺风+大空间+波动小 → 30M
- 逆风反转+波动大+目标近 → 5M
- 顺风但目标近 → 10M
- 挤压突破+强动能 → 30M

# v3.13.4 修复"每次刷新插件都弹上下文失效横幅"

## 问题
在 chrome://extensions 刷新扩展后，页面每次都弹"⚠️ GOODLE 扩展已更新或上下文失效 (storage.get sync throw)"，即使新内容已正常载入。

## 根因
横幅逻辑过于激进：`safeStorageGet/Set/Remove` 的同步 catch 里，**任何**同步异常（包括 content.js 注入时机早于 chrome.storage 就绪、patch 包装的瞬时边角异常）都会弹横幅，把"正常新载入的瞬时抖动"误判成"上下文失效"。

## 修复
1. 同步 catch：只有 `_isCtxInvalidated(e)` 为真（错误信息确含"Extension context invalidated"等）才弹横幅，其余静默 warn。
2. `_showCtxLostBanner` 增加守卫：弹之前检查 `chrome.runtime.id`——真·上下文失效时它才会变 undefined；新页面正常载入时它有值，于是不弹。
3. 结果：扩展更新后只有**真正残留的旧页面**会提示刷新；正常刷新/重载不再误弹。

# v3.13.6 修复"5M在涨却看跌"（动能滞后压过结构）

## 根因
执行周期方向 = 斜率结构S + 动能M + 量能V。原打分有三处让"上涨初段/突破初期"被判看跌：
1. trendBias 只认 bullish/bearish 给分，**mild_bullish/mild_bearish(上涨初段最常见)丢分** → S 被低估。
2. MACD 柱用**绝对符号**(滞后)：上涨初段柱常还是负 → 直接 M-1，与价格方向相反。
3. 动能 M 权重与结构 S 相当，滞后的动能能压过结构定方向——违背 Brooks"结构(always-in)优先"。

## 修复（content.js `_phaseEngine`）
1. trendBias mild 档给分(±1)；新增"价 vs EMA8/EMA21 即时位置"±1，快速反映当下斜向、补结构滞后。
2. MACD 柱权重砍半(±1→±0.5)，降低滞后干扰。
3. **动能与结构冲突时限幅**：S>0 时 M 最多 -1.5，S<0 时 M 最多 +1.5 —— 结构说了算，滞后动能不能把方向带反。

## 离线验证
- 极端初段(结构range/bars mixed/MACD负)：旧 op=-2(偏看跌) → 新 op=1(观望，不再看跌)

- 真下跌：op=-7.5(看跌) ✅ 未误伤

## 重要：诊断可见

# v3.14.0 架构转向：LLM 主导、4 Agent 协作纠错（重大）

## 为什么
之前几版一路加强规则引擎(_phaseEngine)，把方向/相位/期限/空间全算死，再命令 4 个 Agent"以 phase 为准"。结果 LLM 被架空成规则复读机——引擎机械错误(贴阻力追多、滞后指标带反方向、把回撤当反转)时，LLM 跟着错，不会自我纠错。这违背"4 Agent 协作"的本意。

## 改成什么（B 架构：LLM 主导）
1. **规则引擎降为"参考信号"**：相位块措辞从"决策含义/方向=tradeDir/第一权重"改为"规则引擎倾向，仅供参考，可推翻，请结合原始K线自行判断"。
2. **分析师 = 首席判官**：自己看懂趋势结构/动能/量价/关键位空间/节奏，独立得出方向，再对照引擎参考，冲突时以自己分析为准并说明为何推翻。
3. **质疑师 = 独立复核官**：自己也看一遍行情，与分析师"两人独立判断再碰撞"，挑硬伤(贴阻力/无量/背离/接刀)，也列支持事实，给出"成立/存疑/方向可能错"。
4. **裁判 = 终审法官**：独立印象 + 看两人是否一致 → 一致放行、分歧裁断谁证据更扎实、都没把握则观望。规则引擎仅参考。
5. **代码只保留两道硬安全线**：真死盘(15M BBW<0.45% / 5M ATR<12) + 保本线(置信度<~53%)。其余全交给 LLM 判断。
6. 引擎的 no_room/exhaustion 等不再强制覆盖裁判，仅作为相位参考信号提示 LLM。

## 自我提升
随着 autoSessions 数据积累：历史学家提供同类胜率先验、元裁判每30条审计找系统性偏差、偏差记忆注入——4 Agent 在 LLM 自主判断的基础上靠这些反馈持续校准，而不是被死规则锁住。

## 注意
- LLM 主导后，结论不再像纯规则那样"整齐"，不同 Agent 会有分歧——这正是纠错与看懂特殊行情的来源。
- 装 3.14.0 后务必对每个 Agent 点「恢复默认」加载新提示词。
- 模型能力直接影响效果：建议分析师/裁判用较强模型。

# v3.14.1 五本书交易心法注入「交易背景」（4 Agent 共享）

## 为什么
v3.14 转 LLM 主导后，4 Agent 提示词只有"方法/规则"，没有书里的"原理和取舍"。LLM 不知道为什么(如"高周期对剥头皮不利")，遇到新情况无法用书的思想推理，只能机械执行。

## 改成什么（方案B：写进 background，4 Agent 共享，省 token）
`DEFAULT_BACKGROUND` 从一句话扩为含【交易心法·五本书核心】：
1. **价格行为(Brooks)**：always-in 主导方向、回撤结束才入场不接刀、力竭别追最后一棒。
2. **量价(VPA)**：量是因价是果、放量配合才真、缩量/高潮量警示、背离预警。
3. **剥头皮(Volman)**：★在哪个周期交易就以哪个周期为准，高周期只作背景、滞后且制造噪声、**绝不锁死/否决执行周期方向**；20EMA价值线。
4. **做市商(反陷阱)**：显而易见的突破常是陷阱、**勿买在阻力墙下/卖在支撑地板上**、止损猎杀反向、整数关口警惕、逆向拥挤交易。
5. **短线交易**：波动率压缩后扩张顺突破首根、顺动能不接飞刀、严守纪律宁错过不做亏期望单。
+ 期限即时间投影、整体取舍(适度高频 vs 低质量观望)、规则引擎仅参考。

## 生效方式
- 注入分析师/质疑师/裁判(它们都读 background)。
- onInstalled 自动迁移：版本升到 v3.14，未自定义过的 background 自动升级为含心法版；已自定义且含"交易心法"标记的保留。
- 装 3.14.1 后建议在设置确认「交易背景」已含五本书心法；若空白可点恢复默认。

# v3.14.2 期限四档平权（不被"剥头皮"或顺逆背景压短）

## 你的疑问
背景里"剥头皮(Volman)以1M/5M为准"那段，会不会把【到期期限】限制在短周期？

## 澄清 + 修复
1. 那段讲的是【分析周期】(看哪张图判方向)，不是【到期期限】——但和「期限投影」放一起易让 LLM 误解为"剥头皮=只能短到期"。
2. 背景已改：
   - 剥头皮段明确加一句"这是看哪张图判方向，与到期期限是两回事，用短周期图分析不代表只能选短到期"。
   - 「期限即时间投影」段强调 **5/10/15/30 四档完全平权**，按需自由选，不偏向短也不偏向长；方向与期限是独立维度。
3. 代码 `_expiryEngine` 同步：
   - **逆大周期(headwind)不再单独压短**(逆风也可能有大空间)；只有"明确反转/反手单"因节奏快适度偏短(-1.5→-1)。
   - 期限纯由 动能续航 + 波动率 + 前方空间 决定，不因顺逆背景锁定。

## 结果
动能强 + 空间大(到下一阻挡≥3ATR)时，即便是 5M 图分析、即便逆大周期，也能果断给 15/30M 吃整段。期限真正与方向同等独立。
