# v3.13 决策层重建（以 PA 价格行为系统为准绳）

修复目标：解决 v3.12.4 的「分析师 20% 胜率 + 裁判 100% 观望 + 极少给方向」。

## 安装方式
1. Chrome 地址栏打开 `chrome://extensions`
2. 右上角打开「开发者模式」
3. 点「加载已解压的扩展程序」，选择本文件夹 `BTC-Options-Pro-v3_13`
4. 打开 Binance 行情页（www.binance.com），点右下角悬浮按钮使用
5. 在扩展弹窗「设置」里配置模型 API（与旧版一致）

> 升级提示：本版会在安装时**自动**把"未手动改过"的 4 个 Agent 提示词刷新为 v3.13 决策层版本（`promptDefaultsVersion=v3.13`）。若你之前手动改过提示词，会保留你的版本——想用新版请到设置里点「恢复默认」。

## 关键修复（4 处代码 + 4 段提示词）

### 代码门控
1. **死盘否决松绑**（`background.js` applyDecisionProfileGate）
   - 15M 带宽线 `1.2% → 0.45%`，5M ATR `20 → 12`；豁免从"仅趋势回踩"放宽到"任何已确认相位"。
   - 原因：BTC 1m 级 15M 带宽常态 ~0.8%，旧阈值误杀几乎全部行情 = 100% 观望主因。
2. **置信度门槛动态化（G7）**：写死的 `58%` → 按平台赔率 `W*=1/(1+p)+2%` 动态算。
   - 默认 p=0.85 → 门槛 56%；可在 storage.enhancements.payoutRate 配置；下限 54%。
3. **best_candidate 服从相位（G6 初选）**（`content.js`）：力竭/反转相位重罚高分候选（防把追顶/追底排第一），健康回撤奖励长期限。
4. **修复原始包缺陷**：原 v3.12.4 的 background.js 文件被截断（handleMetaJudge 未闭合，service worker 无法加载）——本版已补全函数尾部并修正一处非法正则 `\b?`。

### 4 段默认提示词（决策层重定义）
- **历史学家** = 概率先验器：只给"同相位顺势历史胜率"，不定方向、不建议观望。
- **分析师** = 机会生成器：G1 环境路由（趋势/区间/挤压三剧本）+ G2 方向引擎（只锚 15M/30M 趋势，**禁用 total_score 定向**），偏向行动，只在真震荡/力竭/反转才观望。
- **质疑师** = 陷阱过滤器：只拦 5 类致命陷阱（高潮量/量价背离/止损猎杀/撞关键位/逆势），否则明确"支持放行"。
- **裁判** = 校准决策器：默认放行顺势，唯一否决点，只在三条件（相位压制/方向不明/致命陷阱）观望。

## 上线后怎么看效果
- 「胜率」面板：观望率应从 ~100% 降到 30~45%；分析师胜率回升。
- 按「市场状态/相位」分桶看胜率，长期 <55% 的桶要拉长期限或禁用。
- 每 15 条已验证记录触发元裁判审计，可一键应用其提示词微调建议。

本扩展为基于公开技术分析方法的决策辅助工具，不构成投资建议或盈利保证。所有阈值需结合真实回测持续校准。二元期权为高风险品种，可能损失全部本金，请自负盈亏。

# v3.13.1 相位引擎重写（依五本书核心：趋势/回踩/反转 = 斜率结构+动能+量能 共同决定）

## 修复的核心问题
上涨趋势中分析师/裁判一路看涨、回踩全部亏损。根因：方向被死锁在 15M/30M 高周期，执行周期(5M)反向后无人能否决。

## 改了什么（content.js `_phaseEngine` 全量重写）
- **方向不再死锁高周期**：高周期(15M+30M)=背景偏向(bgDir，顺风/逆风)；执行周期(5M)用 **斜率结构 S + 动能 M + 量能 V 加权投票**(scores.op) 解析"当前可交易方向 tradeDir"。
- **不搞硬共振**(采纳你的意见，避免错过机会)：三维加权打分竞争，谁强归谁。
- **新相位语义**：
  - `retrace_progress` 回踩进行中 → **观望**(等回踩结束，严禁途中接刀，这是回踩亏损的根治)
  - `retrace_end` 回踩结束 → 顺势(短周期可入)
  - `reversal_confirmed` 反转确认 → **顺执行周期方向(可反手)**，依据=结构破位/放量+合力强(有迹可循，非乱反手)
  - `impulse` 推进 / `exhaustion` 力竭(≥2项衰竭) / `unclear` 方向不清
- **方向以 phase.tradeDir 为准**(不是高周期 bgDir)；4 个 Agent 提示词同步：方向取 tradeDir，回踩进行中观望，反转确认跟随。

## 离线验证（构造你截图场景）
- 截图(上涨背景+5M回踩) → 观望 ✅(不再接刀)
- 5M放量破位 → 反手看跌 ✅
- 回踩结束/顺势推进 → 看涨 ✅

# v3.13.2 纯剥头皮模式（Volman：执行周期定方向，大周期不锁死）

## 为什么
剥头皮超短线(1-5M)用 30M/1H 锁方向，书里(Volman/短线交易真好玩/Brooks)是反对的：在哪个周期交易就以哪个周期为 always-in 方向，大周期只作背景。上一版仍让大周期对立时"回踩观望"，对超短线偏慢偏锁。

## 改了什么（content.js `_phaseEngine`）
- **方向 = 执行周期(1M/5M)合力 opDir**：|op|≥2.5 直接定向，顺逆大周期都给。
- **大周期(15M/30M)降级为顺风/逆风**，只调期限与置信，绝不否决：
  - 顺风(tailwind)：impulse，可短打
  - 逆风(headwind)：counter_scalp / reversal_confirmed —— 方向照给，缩 5M 快进快出、可降注
  - 大周期中性：纯看执行周期
- **力竭**：只在 |op|≥4 且伸展到极值(2.0ATR/bbPct≥0.9) + ≥2 衰竭时才短暂观望（门槛高，少误杀）。
- 删除"回踩进行中→观望"逻辑（那是趋势交易思路，不适合剥头皮）。
- 4 Agent 提示词同步：方向取 phase.tradeDir，期限默认 5M，逆大周期照做，严禁因大周期方向否决执行周期信号。

## 离线验证
- 截图场景(大周期涨+5M跌) → **看跌 5M** ✅(不再被锁观望)
- 放量破位 → 看跌 5M(反转确认) ✅
- 顺势上涨 → 看涨 5M ✅

# v3.13.3 动态到期引擎（到期周期与方向同等重要，由书的核心算出，不锁死）

## 为什么
到期周期(赌多久)和方向同等重要，不能默认锁 5M。书的核心(Brooks 测量移动 measured move / Volman 动能续航 / 短线交易波动率)讲的就是：到期时长 = 这段方向"能维持多久/走多远"的时间投影。

## 改了什么（content.js 新增 `_expiryEngine`）
到期周期由 4 个维度动态算出 5/10/15/30：
1. **动能续航**：|op| 合力越强 → 趋势能延续 → 可长
2. **波动率(兑现速度)**：ATR/价格大 → 快速兑现 → 宜短；bbw 极窄(挤压) → 突破后持续 → 略长
3. **到关键位距离(measured move 目标)**：顺势方向前方阻挡近 → 短；空间大(≥3ATR) → 长
4. **入场类型**：反转/逆风 → 快进快出 → 短；顺风推进 → 给时间展开 → 可长；已伸展 → 短

- 删除所有写死的"默认 5M / 倾向 5M"。
- 输出带 `expiryWhy`(到期理由)，面板与提示词都显示依据。
- 4 Agent 提示词同步：期限以 phase.suggestExpiry 为准，可微调一档但须说明依据，严禁无脑锁 5M。

## 离线验证（同样看涨，周期不同）
- 强顺风+大空间+波动小 → 30M
- 逆风反转+波动大+目标近 → 5M
- 顺风但目标近 → 10M
- 挤压突破+强动能 → 30M

# v3.13.4 修复"每次刷新插件都弹上下文失效横幅"

## 问题
在 chrome://extensions 刷新扩展后，页面每次都弹"⚠️ GOODLE 扩展已更新或上下文失效 (storage.get sync throw)"，即使新内容已正常载入。

## 根因
横幅逻辑过于激进：`safeStorageGet/Set/Remove` 的同步 catch 里，**任何**同步异常（包括 content.js 注入时机早于 chrome.storage 就绪、patch 包装的瞬时边角异常）都会弹横幅，把"正常新载入的瞬时抖动"误判成"上下文失效"。

## 修复
1. 同步 catch：只有 `_isCtxInvalidated(e)` 为真（错误信息确含"Extension context invalidated"等）才弹横幅，其余静默 warn。
2. `_showCtxLostBanner` 增加守卫：弹之前检查 `chrome.runtime.id`——真·上下文失效时它才会变 undefined；新页面正常载入时它有值，于是不弹。
3. 结果：扩展更新后只有**真正残留的旧页面**会提示刷新；正常刷新/重载不再误弹。
