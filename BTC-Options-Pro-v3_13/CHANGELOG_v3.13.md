# v3.13 决策层重建（以 PA 价格行为系统为准绳）

修复目标：解决 v3.12.4 的「分析师 20% 胜率 + 裁判 100% 观望 + 极少给方向」。

## 安装方式
1. Chrome 地址栏打开 `chrome://extensions`
2. 右上角打开「开发者模式」
3. 点「加载已解压的扩展程序」，选择本文件夹 `BTC-Options-Pro-v3_13`
4. 打开 Binance 行情页（www.binance.com），点右下角悬浮按钮使用
5. 在扩展弹窗「设置」里配置模型 API（与旧版一致）

> 升级提示：本版会在安装时**自动**把"未手动改过"的 4 个 Agent 提示词刷新为 v3.13 决策层版本（`promptDefaultsVersion=v3.13`）。若你之前手动改过提示词，会保留你的版本——想用新版请到设置里点「恢复默认」。

## 关键修复（4 处代码 + 4 段提示词）

### 代码门控
1. **死盘否决松绑**（`background.js` applyDecisionProfileGate）
   - 15M 带宽线 `1.2% → 0.45%`，5M ATR `20 → 12`；豁免从"仅趋势回踩"放宽到"任何已确认相位"。
   - 原因：BTC 1m 级 15M 带宽常态 ~0.8%，旧阈值误杀几乎全部行情 = 100% 观望主因。
2. **置信度门槛动态化（G7）**：写死的 `58%` → 按平台赔率 `W*=1/(1+p)+2%` 动态算。
   - 默认 p=0.85 → 门槛 56%；可在 storage.enhancements.payoutRate 配置；下限 54%。
3. **best_candidate 服从相位（G6 初选）**（`content.js`）：力竭/反转相位重罚高分候选（防把追顶/追底排第一），健康回撤奖励长期限。
4. **修复原始包缺陷**：原 v3.12.4 的 background.js 文件被截断（handleMetaJudge 未闭合，service worker 无法加载）——本版已补全函数尾部并修正一处非法正则 `\b?`。

### 4 段默认提示词（决策层重定义）
- **历史学家** = 概率先验器：只给"同相位顺势历史胜率"，不定方向、不建议观望。
- **分析师** = 机会生成器：G1 环境路由（趋势/区间/挤压三剧本）+ G2 方向引擎（只锚 15M/30M 趋势，**禁用 total_score 定向**），偏向行动，只在真震荡/力竭/反转才观望。
- **质疑师** = 陷阱过滤器：只拦 5 类致命陷阱（高潮量/量价背离/止损猎杀/撞关键位/逆势），否则明确"支持放行"。
- **裁判** = 校准决策器：默认放行顺势，唯一否决点，只在三条件（相位压制/方向不明/致命陷阱）观望。

## 上线后怎么看效果
- 「胜率」面板：观望率应从 ~100% 降到 30~45%；分析师胜率回升。
- 按「市场状态/相位」分桶看胜率，长期 <55% 的桶要拉长期限或禁用。
- 每 15 条已验证记录触发元裁判审计，可一键应用其提示词微调建议。
