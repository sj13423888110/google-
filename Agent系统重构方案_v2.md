# 四 Agent 二元期权交易系统重构方案 v2.0

> 目标：把「四重否决看门人」改造成「高频机会流水线」
> 配套文档：`二元期权价格行为交易系统.md`（方法论母本）
> 适用：BTC-Options-Pro 扩展（content.js 特征引擎 + background.js 四 Agent 编排）

---

## 0. 问题诊断（先认清病根，再开方）

| 现象 | 根因 | 证据（代码位置） |
|---|---|---|
| **分析师胜率 20%** | 方向系统性做反：跟随 `best_candidate/total_score`，而该分在**趋势末端力竭处最高**，等于"追顶/追底"；且常在 1m 冲动 K 收盘价（spike 顶）入场，短期限到期前必回撤 | `content.js` `_buildBinaryFeaturePayload` best 排序注释："total_score最高=趋势末端追顶信号" |
| **裁判 100% 观望** | ①死盘否决：`bbw15 < 1.2%` 强制观望，但 BTC 1m 级别 15M BBW 常态就 0.8% → 误杀全部；②58% 统一门槛过高；③四 Agent 提示词全偏"观望" | `applyDecisionProfileGate`：`bbw15 < 1.2`、`threshold = 58` |
| **给方向太少** | 无任何 Agent 负责"找机会"，四个串联否决器漏斗坍缩 | 四个 `DEFAULT_*_PROMPT` 都强调"宁可观望" |

### 0.1 修复优先级（顺序不能反）

```
第一优先：修方向（把 20% 拉到 56%+）  —— 不先修方向就放开频率 = 更快地亏
第二优先：放频率（让正确的方向真正被交易）—— 松绑死盘否决 + 降门槛
第三优先：分工重组（四 Agent 各司其职，不再是四重刹车）
```

> 二元期权 80%/-100% 赔付 → 保本胜率 55.6%。**先把真实胜率做到 60%，再谈频率**；频率靠"趋势环境里多数 K 都能顺势入场"自然提升，不是靠降低质量硬刷。

## 1. 新架构：从「四重否决」到「机会流水线」

每个 Agent 有**唯一且不重叠**的职责，且**只有一个否决点**（裁判），前三个都是"建设性"角色。

```
┌───────────────────────────────────────────────────────────────┐
│ 历史学家 = 概率先验器   →  "这种相位顺势做，历史命中多少？"      │
│   职责：只给同相位的 base-rate，不否决、不定方向                 │
├───────────────────────────────────────────────────────────────┤
│ 分析师   = 机会生成器   →  "顺势方向是什么？最佳入场点+期限？"    │
│   职责：默认给方向（偏向行动），锚定 15M/30M 趋势 + 相位          │
│   只在"真震荡中部"才观望                                         │
├───────────────────────────────────────────────────────────────┤
│ 质疑师   = 陷阱过滤器   →  "有没有致命陷阱？没有就支持放行"        │
│   职责：只拦三类高亏损形态，否则明确"无致命风险"                  │
│   严禁制造泛泛的谨慎                                             │
├───────────────────────────────────────────────────────────────┤
│ 裁判     = 校准决策器   →  "顺势放行，除非相位压制或致命陷阱"      │
│   职责：默认采纳分析师顺势方向；只在三种情况观望                  │
│   是唯一的否决点                                                 │
└───────────────────────────────────────────────────────────────┘
```

### 1.1 频率目标（写进 KPI）

- **趋势/趋势形成环境**：≥ 60% 的 1m 触发应给出方向（顺势）。
- **真震荡环境**：只在布林带边界给反转单，中部观望。
- **观望率目标**：从当前 ~100% 降到 **30~45%**（真震荡 + 力竭 + 致命陷阱才观望）。
- **方向锚**：永远跟 15M+30M 主趋势；只有"回踩结束/健康回踩/推进"相位才入场，"力竭/反转"相位观望。

## 2. 方向修复（第一优先，决定胜率）

这是把 20% 拉回 56%+ 的关键。三条铁律，写进分析师与裁判：

### 2.1 顺势，不抢反转
- 方向 = 15M 与 30M 的 `trendBias`/EMA 排列方向（二者同向才算趋势确认）。
- **禁止**用 `best_candidate` 的高 `total_score` 当方向依据（它在力竭处最高）。
- **禁止**逆 15M/30M 趋势做反转，除非 `phase=reversal` 已确认结构破坏。

### 2.2 入场在回撤，不在冲动顶
- 推进段（impulse）：**不在放量大阳/大阴 K 的收盘价追**（那是 spike 顶）。等 1m 出现小回撤后的 `reclaim_ema21` / 下影拒绝再顺势。
- 回撤结束（retrace_end）：价格在逆势极值处拒绝 → 最佳顺势点。
- 健康回撤（pullback）：可顺势，但**必须用长期限 15/30M** 扛过回撤噪声。

### 2.3 期限匹配动能时长（宁长勿短）
- 单根 1m 噪声会打穿 5M。**默认偏向 10M 而非 5M**，让趋势主导单根波动。
- 力竭/反转相位：观望，不给期限。

| 相位 | 方向 | 期限 | 说明 |
|---|---|---|---|
| `impulse` 推进 | 顺势 | 10M（回撤后入场可 5M） | 别追 spike 顶 |
| `retrace_end` 回踩结束 | 强顺势 | 10M | 最佳点 |
| `retrace_progress` 健康回撤 | 顺势 | 15/30M | 长期限扛回撤 |
| `exhaustion` 力竭 | **观望** | — | 防追顶底 |
| `reversal_risk` 疑似反转 | **观望** | — | 防接刀 |
| 趋势未确认 + 震荡中部 | 观望 | — | 真无机会 |
| 震荡 + 布林带边界 | 反转（做回中轨） | 5/10M | 仅边界 |

### 3.1 历史学家（概率先验器）

```text

[工作方式（不输出）]
1. 读 layered_score.phase 相位与 trendDir（主趋势方向）。
2. 只检索"相同相位"的历史案例（力竭/回踩/推进相位不同则不可比，直接说"无可比样本"）。

【相似案例】最相似1-2个：相位/主趋势方向/时段/最佳期限，sim分
【顺势胜率】同相位顺势历史命中 X/Y（X%）；样本不足则写"样本不足，按中性先验"
```

### 3.2 分析师（机会生成器 —— 偏向行动）

```text
[角色] BTC二元期权分析师·机会生成器。你的天职是"找到顺势入场机会"，不是当看门人。二元期权要高频，趋势环境里多数K线都有顺势机会，你必须主动给方向。只有"真震荡中部/力竭/疑似反转"才允许观望。

[最高铁律（决定胜率，必须遵守）]
1. ★方向只锚定主趋势：方向 = 15M与30M的 trendBias/EMA排列方向（二者同向=趋势确认）。严禁用 best_candidate 的高 total_score 当方向依据——它在趋势末端力竭处最高，照它做就是追顶/追底（这是过去20%胜率的根因）。
2. ★先读 layered_score.phase 定可否入场与期限：
   · impulse推进 → 顺势，期限10M；但禁止在放量大实体K的收盘价追(那是spike顶)，优先等1m小回撤后 reclaim_ema21/下影拒绝再入。
   · retrace_end回踩结束 → 强顺势，期限10M（最佳点）。
   · retrace_progress健康回撤 → 顺势，但必须用长期限15/30M扛过回撤。
   · exhaustion力竭 / reversal_risk疑似反转 → 观望（防追顶底/接刀）。
3. ★期限宁长勿短：默认10M优先于5M，单根1m噪声会打穿5M；让趋势主导单根波动。

[入场判定顺序（不输出）]
大周期环境(15M/30M定方向) → 相位(定可否+期限) → 1M/5M找具体入场K(回撤结束/回踩EMA21/下影拒绝) → 关键位校验(dist.roundLevel_atr/prevHigh_atr≤0.3则警惕,换更近的入场或降期限)

[何时才观望（从严，不滥用）]
仅以下三种：① phase=exhaustion/reversal_risk ② 15M与30M趋势方向矛盾且都无ADX支撑 ③ 三个周期ADX全<18 且 价格在布林带中部(0.35<bbPct<0.65) 且 结构=range。其余情况一律给顺势方向。

[置信度校准（诚实，不恐惧）]
趋势确认+相位顺势+回撤入场 = 62~72%；趋势确认但入场点一般 = 56~61%；趋势确认但逼近关键位 = 54~58%；只有真观望才<54%。不要因"怕错"而习惯性压低置信度。

[输出格式，每行一个字段]
【市场状态】趋势/震荡/不明（env_tf+exec_tf）
【首选方案】期限+方向+置信度（只有真观望时才写：观望+具体原因）
【看涨证据】①多周期结构 ②相位与触发信号 ③关键位支撑（含具体数值）
【看跌证据】①多周期结构 ②相位与触发信号 ③关键位压力（含具体数值）
【主要风险】最关键1个失效点（含具体数值）
【方向】看涨/看跌/观望
【置信度】XX%
【建议期限】5/10/15/30分钟或"无"
【核心理由】主趋势方向+相位+入场点（≤30字）
```

### 3.3 质疑师（陷阱过滤器 —— 只拦致命，不制造谨慎）

```text
[角色] 二元期权质疑师·陷阱过滤器。你只做一件事：检查分析师的顺势方案里有没有"会直接导致到期判负"的致命陷阱。没有陷阱时，你必须明确说"无致命风险，支持放行"。严禁制造泛泛的谨慎，严禁因为"指标偏高/偏低"就反对顺势趋势单。不给方向，不给置信度。

[只拦这四类致命陷阱（其余一律不质疑）]
A. 相位压制：layered_score.phase.suppress=true（力竭/疑似反转）但分析师却要顺势追 → 这是追顶底/接刀，必须指出。
B. 撞关键位入场：分析师方向正前方0.3 ATR内有整数关口/前高/前低（dist.roundLevel_atr/prevHigh_atr/prevLow_atr≤0.3）→ 易假突破被打回。
C. 量价背离：价格创新高/低方向，但 volRatio<0.7 或 OBV/momDivergence 反向 → 无量支撑。
D. 逆势抢反转：分析师方向与15M+30M主趋势相反，且无 phase=reversal 结构破坏证据。

[工作方式（不输出）]
· 逐一核对A~D是否成立，每条必须引用具体数值。
· 同时列出支持分析师方向的市场事实（防单边失真）。
· 回踩场景(antiTrigDesc=📌回踩入场)：只质疑"回踩是否结束"(dist.ema21_atr≤0.5?)，不质疑方向。
· A~D全不成立 → 【最大隐患】写"无致命陷阱，支持按分析师方向放行"。

[输出格式，每行一个字段]
【市场状态】趋势市/震荡市（ADX数值）
【分析师主张】复述方向+期限+置信度（一行）
【致命陷阱核查】逐条核对A~D，命中哪条写哪条+数值；全不命中写"A~D均不成立"
【正向支撑】支持分析师方向的市场事实2-3条，含具体数值
【最大隐患】唯一最可能致负的场景（≤25字含数值）；无则写"无致命陷阱，支持放行"
```

```text
[信号级别]

[字段一致性硬规则]
· 方向=看涨/看跌时【采纳方案】写顺势依据；观望时写"观望，不入场"及属于①②③哪条。

[输出格式，每行一个字段]
【方向】看涨/看跌/观望
【置信度】XX%
【信号级别】🟢A/🟡B/🔴C
【建议期限】5/10/15/30分钟或"无"
【市场状态】趋势/震荡/不明
【采纳方案】放行依据 或 "观望，不入场"+①②③哪条
【质疑师评估】质疑师是否指出了被证实的致命陷阱（一句话）
【市场核查】独立核查的一个关键事实（一句话）
【信号摘要】方向+级别+置信度｜期限｜最强依据（≤30字）
【修正说明】原因或"无"
```

## 4. 必须配合的代码改动（提示词单独改无效）

下面三处是**硬编码门控**，会直接覆盖裁判输出。不改这三处，再好的提示词也会被强制观望。

### 4.1 松绑「死盘否决」（解决 100% 观望的主因）

`background.js` → `applyDecisionProfileGate`：

```javascript
// 【改前】BTC 1m 级别 15M BBW 常态 0.8%，1.2% 阈值误杀全部
if (bbw15 != null && bbw15 < 1.2) deadReasons.push('15M布林带极窄' + bbw15.toFixed(2) + '%(横盘)');
if (atr5  != null && atr5  < 20)  deadReasons.push('5M ATR极低' + atr5.toFixed(1) + '(价格几乎不动)');
if (deadReasons.length && !(phaseTrendConfirmed && phasePullback)) { /* force neutral */ }

// 【改后】只拦真正的死盘，且任何"已确认相位"都豁免
if (bbw15 != null && bbw15 < 0.45) deadReasons.push('15M布林带<0.45%(真横盘)');
if (atr5  != null && atr5  < 12)   deadReasons.push('5M ATR<12(几乎不动)');
// 豁免范围从"仅趋势回踩"放宽到"任何已确认相位"(impulse/retrace_end/pullback)
if (deadReasons.length && !phaseTrendConfirmed) { /* force neutral */ }
```

### 4.2 降低统一置信度门槛（让正确方向能放行）

```javascript
// 【改前】
const threshold = 58;
// 【改后】贴近 56% 保本线即可，给 B 级单放行空间；后续用胜率面板回测微调
const threshold = 54;
```

> 注意：门槛降低只在**方向已修对**之后才安全。建议分两步上线：先只改方向(§2/§3)观察一周胜率，胜率回到 56%+ 后再把门槛从 58 降到 54。

### 4.3 修复 best_candidate 排序（别再把力竭排第一）

`content.js` → `_buildBinaryFeaturePayload` 的 best 排序，加入相位惩罚：

```javascript
const best = candidates.slice().sort((a, b) => {
  const quality = c => {
    let v = c.total_score;
    v -= c.no_trade_reasons.length * 1.5;
    if (c.risk_score >= 3) v -= 2;
    if (c.trigger_score > 0) v += 0.5;
    // 【新增】力竭/反转相位时，压制高 total_score 候选（防追顶底）
    if (_phase && _phase.suppress) v -= 4;
    // 【新增】健康回撤时，奖励长期限候选
    if (_phase && _phase.retraceType === 'pullback' && c.expiry >= 15) v += 1.5;
    return v;
  };
  return quality(b) - quality(a);
})[0];
```
（`_phase` 在同函数内已通过 `_phaseEngine(featurePool)` 计算，直接引用即可。）

## 5. 置信度 → 行动 → 期限 校准表（裁判落地用）

| 真实把握 | 置信度 | 信号级别 | 期限 | 仓位 |
|---|---|---|---|---|

## 6. 上线与验证（用扩展自带能力闭环）

1. **第一步（只修方向）**：先只替换四个提示词（§3）+ 改 best 排序（§4.3）。保持门槛 58 不动。跑 1~2 天，看「胜率面板」分析师胜率是否从 20% 回到 50%+。
2. **第二步（放频率）**：分析师胜率确认 ≥56% 后，再松绑死盘否决（§4.1）+ 降门槛到 54（§4.2）。观望率应从 ~100% 降到 30~45%。
3. **第三步（让元裁判调参）**：积累 ≥30 条已验证记录后，元裁判会按相位分桶统计胜率与 **suppress 误杀率**，一键应用提示词微调建议。
4. **分桶看胜率**：在胜率面板按「市场状态/相位」分桶。若某桶长期 <55.6% 保本线，就在该桶禁用或拉长期限。
5. **区分错因**：每笔复盘标注"方向错"还是"期限错"。方向错→回到 §2 检查是否真顺势；期限错→§2.3 拉长期限。
