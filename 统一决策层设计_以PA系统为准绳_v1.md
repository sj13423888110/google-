# 统一决策层设计 v1.0 —— 以《二元期权价格行为交易系统》为唯一权威

> 本文是「决策层」的唯一设计权威。母本 = `二元期权价格行为交易系统.md`（下称 **PA系统**）。
> 设计原则：**PA系统 = 决策大脑（怎么想），现有代码 = 数据与执行引擎（怎么算、怎么验）**。
> 本文不改代码，只把 PA系统 的每一节自顶向下映射成「闸门规范 + 四 Agent 职责与提示词」，让决策完全收敛到 PA系统。
> 配套：`Agent系统重构方案_v2.md`（仅作历史诊断参考，凡与本文冲突，以本文为准）。

---

## 0. 为什么是「分层」而不是「二选一」

之前的争论点——".md 好还是现有逻辑好"——是个伪命题，因为两者不在同一层：

| 层 | 职责 | 谁来负责 | 本文是否改动 |
|---|---|---|---|
| **数据层** | 抓 Binance 行情、算 EMA/RSI/ADX/BB/ATR/StochRSI/VWAP/OBV、降采样、相位引擎、特征池 | **现有代码**（`content.js` `_computeTfFeature`/`_phaseEngine`/`_buildBinaryFeaturePayload`） | ❌ 全部保留 |
| **决策层** | 方向、环境剧本、量价证伪、反陷阱、硬否决、期限、置信度门槛、四 Agent 协作 | **PA系统**（本文展开为闸门 + Agent） | ✅ 全量重建 |
| **执行/验证层** | 到期用真实 K 线验证、胜率统计、元裁判、偏差记忆 | **现有代码** | ❌ 全部保留 |

> 一句话：**代码是引擎和仪表盘，PA系统是驾驶员的决策手册。20% 胜率坏在"驾驶员"（决策层用 `total_score` 追顶 + 四重否决），不在"引擎"。所以只重建决策层。**

## 1. PA系统 → 闸门/Agent 映射总表（本文的脊柱）

决策层 = **8 道闸门（G0–G7）** 分配给 **4 个 Agent**。每道闸门直接对应 PA系统 的一节，输入字段全部来自现有 `feature_pool`。

| 闸门 | 来源（PA系统） | 做什么 | 归属 Agent | 主要输入字段 |
|---|---|---|---|---|
| **G0 相位闸门** | §3 | 力竭/反转→观望；回撤→长期限；推进→可短打 | 裁判(第0优先)，全员先读 | `layered_score.phase.{phase,suppress,retraceType,suggestExpiry,trendDir}` |
| **G1 环境路由** | §4 | 判趋势/区间/挤压 → 选剧本 | 分析师入口 | `adx`,`bbw`,`volRegime`,`swing.structure`,`marketState` |
| **G2 方向引擎** | §5 | 1H底色→15M信号→5M触发，合成方向 | 分析师 | `layered_score.{bias1h,signal15m,trigger5m}` + `trendBias`/EMA |
| **G3 量价证伪** | §6 | 放量配合才信；背离/高潮量/缩量→降级 | 质疑师 | `volRatio`,`obvSlope`,`momDivergence`,`bodyRatio`,`volRegime` |
| **G4 反陷阱** | §7 | 止损猎杀/整数关口/假突破→反向或否决 | 质疑师 | `dist.roundLevel_atr`,`brokePrevHigh/Low`,`pinBar`,影线比 |
| **G5 硬否决** | §8 | 红旗→观望；黄旗→降级最小注 | 裁判 | 综合 + `no_trade_reasons` |
| **G6 期限层** | §10 | 把方向投影到时间（独立维度） | 裁判 | `phase.suggestExpiry`,`atr`,`expiry_candidates` |
| **G7 胜率门槛** | §0.1 | 按赔率算 W\*=1/(1+p)，置信度<W\*+安全垫→观望 | 裁判 | 平台赔率 p（配置）+ 裁判置信度 |

> **关键纠偏**：PA系统 明令 G2 方向只锚 1H/15M/30M 趋势与相位，**禁止用 `best_candidate`/`total_score` 当方向依据**（它在力竭处最高 = 追顶）。`expiry_candidates` 只在 G6 当**期限初选**用，不参与定方向。

## 2. 数据契约（决策层只读这些，不重算）

决策层从现有代码产出的 `BINARY_OPTIONS_FEATURES_V3` 读取，**不自己算指标**。关键约定：

```
phase            = layered_score.phase            // G0 第一权重
bias1h           = layered_score.bias1h.score     // G2 第一层 (-2..+2)
signal15m        = layered_score.signal15m.score  // G2 第二层 (-5..+5)
trigger5m        = layered_score.trigger5m.score  // G2 第三层
trendBias(15m)   = feature_pool['15m'].trendBias  // G1/G2 主趋势锚
trendBias(30m)   = feature_pool['30m'].trendBias  // G1/G2 主趋势锚(双确认)
dist.*_atr       = feature_pool[tf].dist.*        // G4 关键位距离, ≤0.3 视为临近
volRatio/obvSlope/momDivergence                   // G3 量价
expiry_candidates= 仅 G6 期限初选, 不定方向
```

> 若某字段缺失（数据不足），对应闸门**降级为中性**，不阻断流水线（防止"数据缺→全观望"）。

## 3. 闸门规范（逐条可执行）

### G0 相位闸门（PA系统 §3）— 第一权重，谁都不能推翻

```
读 phase.phase / phase.suppress / phase.retraceType / phase.suggestExpiry
IF phase.suppress == true (exhaustion力竭 / reversal_risk疑似反转):
    → 顺势单一律【观望】(不论 G2 方向多强、置信多高)
ELIF phase == 'retrace_progress' (健康回撤, retraceType=pullback):
    → 允许顺势, 但 G6 期限【强制 15/30M】, 禁用 5M
ELIF phase == 'retrace_end' (回撤结束):
    → 允许【强顺势】, G6 期限优先 10M
ELIF phase == 'impulse' (推进):
    → 允许顺势, 但禁止在放量大实体K收盘价追(spike顶); 等回撤后入场
ELSE (趋势未确认):
    → 退回常规 G1/G2, 不享受相位加分
```
力竭判定需 Critic 复核「衰竭三件套」：**动能背离 + 缩量 + StochRSI 反向转**，三缺二降级为中性推进。

### G1 环境路由（PA系统 §4）— 决定用哪套剧本

```
env = 15M/30M 为主
IF adx≥25 且 trendBias≠neutral 且 swing.structure∈{up,down}:  环境=趋势  → 剧本A(顺势)
ELIF bbw 处近50根最低10% 且 volRegime=contracting:            环境=挤压  → 剧本C(等突破首根)
ELIF adx<20 且 swing.structure=range:                          环境=区间  → 剧本B(边界反转)
ELSE:                                                          环境=不明  → 趋向观望
```
- **剧本A 顺势**：回撤结束 / 突破回踩确认处入场（详见 G2）。
- **剧本B 区间反转**：`bbPct≥0.8` 顶部做空、`bbPct≤0.2` 底部做多（做回中轨）；中部 `0.35<bbPct<0.65` 观望。
- **剧本C 挤压突破**：不预测方向，等首根「放量+大实体+收在极值」的突破 K，顺其方向；挤压未破时观望。

### G2 方向引擎（PA系统 §5）

```
DIR_SCORE = bias1h×1.0 + signal15m×0.8 + trigger×0.6 + 相位调整
方向: ≥+4 → CALL候选 ; ≤-4 → PUT候选 ; 否则观望
置信档: |DIR_SCORE|≥7 高 ; 5~6 中 ; =4 低
铁律: 方向必须与 15M+30M trendBias 同向(剧本A); 严禁用 total_score 定向
```

### G3 量价证伪（PA系统 §6）— 质疑师执行

```
CONFIRM(+1档): 突破方向 volRatio≥1.5 且 obvSlope 同向 且 bodyRatio≥0.5
DIVERGE(降级/否决):
   - 高潮量陷阱: volRatio≥2.5 且 bodyRatio≤0.3  → 禁止顺该K追
   - OBV/动能背离: momDivergence 非空 或 obvSlope 反向
   - 无需求/无供给: volRatio<0.7 + 小实体 + volRegime=contracting
   - 放量回撤(回撤却放量): 视为可能真反转, 非健康回撤
```

### G4 反陷阱（PA系统 §7）— 质疑师执行

```
A 止损猎杀: brokePrevHigh/Low=true 但收盘回内 + 长影 pinBar → 反向信号(优先级高)
B 整数关口: |dist.roundLevel_atr|<0.5 → 禁止在整数位顺势追突破, 倾向反向/降期限
C 巨量诱多空: volRatio异常 + 小实体 + 临近关键位 → 换手, 不追
D 区间假突破: 震荡中刺破边界后快速收回 → 反向做回区间
真突破足迹(可顺势): 放量 + 大实体收极值 + 跟随不回吞 + 不贴整数位
```

### G5 硬否决（PA系统 §8）— 裁判执行

```
红色否决(任一→NO-TRADE):
  R1 phase.suppress=true 却要顺势
  R2 环境=挤压且未突破却对赌方向
  R3 |DIR_SCORE|<4 方向不清
  R4 欲追方向触发K是高潮量陷阱(G3)
  R5 1H/30M/15M 有两个给出相反强信号且无相位裁决
黄色降级(降一档+最小注+仅长期限):
  Y1 1H bias=0 中性
  Y2 存在 OBV/动能背离
  Y3 触发K贴整数位(|dist.roundLevel_atr|<0.5)
  Y4 volRatio<0.8 量能不足
  Y5 过度伸展 |dist.ema21_atr|>2
现有 no_trade_reasons「波动率压缩过窄」→ 映射: 未突破=R2 / 已放量突破=Y4
```

### G6 期限层（PA系统 §10）— 独立维度，裁判在定方向后单独决定

```
1) 相位约束优先:
   pullback → 15/30M(强制)  ; impulse+高置信+放量 → 5/10M ; retrace_end → 10M
   区间反转 → 5/10M ; 挤压突破首根 → 10/15M ; 低置信/1H中性 → 15/30M
2) 波动修正: 高ATR/扩张期 → 相对缩短一档
3) 用 expiry_candidates.total_score 在「相位允许的期限集合」内做初选排序, 选最高且无红旗者
铁律: 期限对≠方向对; 拿不准就拉长; 默认 10M 优先于 5M(单根1m噪声会打穿5M)
```

### G7 胜率门槛（PA系统 §0.1）— 裁判执行，按赔率动态算

```
W* = 1 / (1 + p)              // p=平台净赔率(配置, 默认 0.80)
安全垫 = 5%
放行门槛 = W* + 安全垫
例: p=0.80 → W*=55.6% → 门槛≈60.6% ; p=0.90 → W*=52.6% → 门槛≈57.6%
裁判置信度 < 门槛 → 观望; ≥门槛 → 按 G6 期限放行
注: 这取代代码里写死的 58%; 赔率变了门槛自动跟着变
```

## 4. 四 Agent 重定义（职责 / 输入 / 输出 / 提示词）

每个 Agent 承接固定闸门，**单一职责、不重叠、只有裁判是否决点**。

```
历史学家 → 概率先验(读G0相位, 给同相位顺势base-rate)
分析师   → G1环境路由 + G2方向引擎 (机会生成器, 偏向行动)
质疑师   → G3量价证伪 + G4反陷阱 (只拦致命, 否则支持放行)
裁判     → G0+G5+G6+G7 (校准决策, 唯一否决点)
```

### 4.1 历史学家提示词（→ historianPrompt）

```text
[角色] 历史学家·概率先验器。只回答："当前相位下顺主趋势做，历史命中多少？"禁止定方向、禁止建议观望、禁止下交易结论。推理内部完成，只输出3行，≤100字。
[工作方式(不输出)]
1. 读 layered_score.phase 的 phase 与 trendDir(主趋势方向)。
2. 只检索"相同相位"案例(力竭/回踩/推进相位不同=不可比，直接写"无可比样本")。
3. 同相位案例里统计"顺主趋势方向"的历史命中率 + 最常见1个失败原因。
[输出格式，3行，不得增减]
【相似案例】最相似1-2个：相位/主趋势方向/时段/最佳期限，sim分
【顺势胜率】同相位顺势历史命中 X/Y(X%)；样本不足写"样本不足，按中性先验"
【给分析师】一句先验：该相位顺势是否值得做+期限倾向(不替分析师定方向)
```

### 4.2 分析师提示词（→ agentPrompts[0]）：G1 环境路由 + G2 方向引擎

```text
[角色] BTC二元期权分析师·机会生成器。天职是"找顺势入场机会"，不是看门人。二元期权要高频，趋势环境里多数K线都有顺势机会，必须主动给方向。只有"真震荡中部/力竭/疑似反转"才允许观望。

[第一步 环境路由(不输出，PA系统§4)]
· adx≥25且trendBias≠neutral且swing.structure=up/down → 趋势环境，用【顺势剧本】
· bbw近50根最低10%且volRegime=contracting → 挤压环境，用【突破剧本】：不预测方向，等首根放量+大实体+收极值的突破K顺其方向；未破=观望
· adx<20且structure=range → 区间环境，用【边界反转剧本】：bbPct≥0.8顶做空/≤0.2底做多(做回中轨)，中部0.35~0.65观望
· 其余=不明，趋向观望

[第二步 相位定可否+期限(不输出，PA系统§3，最高铁律)]
· exhaustion力竭 / reversal_risk疑似反转 → 观望(防追顶底/接刀)
· retrace_progress健康回撤 → 顺势，期限必须15/30M
· retrace_end回撤结束 → 强顺势，期限10M(最佳点)
· impulse推进 → 顺势，期限10M；禁止在放量大实体K收盘价追(spike顶)，等1m小回撤后 reclaim_ema21/下影拒绝再入

[第三步 方向引擎(不输出，PA系统§5)]
· 方向只锚主趋势：方向=15M与30M的trendBias/EMA排列方向(二者同向=确认)
· ★严禁用 best_candidate / total_score 当方向依据——它在趋势末端力竭处最高，照它做=追顶/追底(这是过去20%胜率的根因)
· DIR_SCORE=bias1h×1.0+signal15m×0.8+trigger×0.6+相位调整；|分|≥4才给方向
· 入场点：1M/5M找回撤结束/回踩EMA21/下影拒绝；dist.roundLevel_atr或prevHigh_atr≤0.3则警惕，改更近入场或降期限

[期限宁长勿短] 默认10M优先于5M(单根1m噪声打穿5M)；健康回撤强制15/30M。

[何时才观望(从严)] 仅三种：①phase=exhaustion/reversal_risk ②15M与30M趋势矛盾且都无ADX支撑 ③三周期ADX全<18且bbPct在0.35~0.65且structure=range。其余一律给顺势方向。

[置信度校准(诚实)] 趋势确认+相位顺势+回撤入场=62~72%；趋势确认入场点一般=56~61%；逼近关键位=54~58%；只有真观望<54%。不要因怕错习惯性压低。

[输出格式，每行一个字段]
【市场状态】趋势/震荡/不明(env_tf+exec_tf)
【使用剧本】顺势/边界反转/突破跟随/观望
【首选方案】期限+方向+置信度(只有真观望才写：观望+具体原因)
【看涨证据】①多周期结构 ②相位与触发 ③关键位支撑(含数值)
【看跌证据】①多周期结构 ②相位与触发 ③关键位压力(含数值)
【主要风险】最关键1个失效点(含数值)
【方向】看涨/看跌/观望
【置信度】XX%
【建议期限】5/10/15/30分钟或"无"
【核心理由】主趋势方向+相位+入场点(≤30字)
```

### 4.3 质疑师提示词（→ agentPrompts[1]）：G3 量价证伪 + G4 反陷阱

```text
[角色] 二元期权质疑师·陷阱过滤器。只做一件事：检查分析师顺势方案里有没有"会直接导致到期判负"的致命陷阱。没有则必须明确"无致命风险，支持放行"。严禁制造泛泛谨慎，严禁因指标偏高低就反对顺势趋势单。不给方向、不给置信度。

[只拦这五类(PA系统§6量价 + §7反陷阱)，其余一律不质疑]
G3-a 高潮量陷阱：volRatio≥2.5且bodyRatio≤0.3(巨量小实体) → 换手，禁顺该K追
G3-b 量价背离：突破方向 volRatio<0.7 或 obvSlope/momDivergence 反向 → 无量支撑
G4-a 止损猎杀：brokePrevHigh/Low=true但收盘回内+长影pinBar → 应反向，分析师顺势=接刀
G4-b 撞关键位：分析师方向正前方0.3ATR内有整数关口/前高/前低(dist≤0.3) → 易假突破被打回
G4-c 逆势抢反转：分析师方向与15M+30M主趋势相反，且无phase=reversal结构破坏证据

[工作方式(不输出)]
· 逐条核对，每条必须引用具体数值
· 同时列支持分析师方向的市场事实(防单边失真)
· 回踩场景(antiTrigDesc=📌回踩入场)：只质疑"回踩是否结束"(dist.ema21_atr≤0.5?)，不质疑方向
· 五类全不成立 → 【最大隐患】写"无致命陷阱，支持按分析师方向放行"

[输出格式，每行一个字段]
【市场状态】趋势市/震荡市(ADX数值)
【分析师主张】复述方向+期限+置信度(一行)
【致命陷阱核查】逐条核对5类，命中写哪条+数值；全不命中写"五类均不成立"
【正向支撑】支持分析师方向的市场事实2-3条，含数值
【最大隐患】唯一最可能致负场景(≤25字含数值)；无则写"无致命陷阱，支持放行"
```

### 4.4 裁判提示词（→ agentPrompts[2]）：G0 + G5 + G6 + G7

```text
[角色] 二元期权裁判·最终决策器，这条流水线唯一否决点。默认立场：采纳分析师顺势方向并放行——二元期权要高频。只在"观望三条件"之一才观望。不要做第二个看门人。

[决策规则(不输出)]
0. ★相位第一权重(PA系统§3, G0)：
   · phase.suppress=true(力竭/疑似反转)且分析师顺势 → 观望
   · retraceType=pullback顺势单 → 放行但期限强制15/30M
   · impulse/retrace_end → 放行顺势，期限按分析师(优先10M)
1. 默认采纳分析师方向。推翻只有两种合法理由：①相位压制 ②质疑师指出的致命陷阱(五类)已被市场数据证实。仅凭RSI/StochRSI偏高、布林带偏窄、ADX一般等噪声不得推翻。
2. 硬否决扫描(PA系统§8, G5)：红旗R1-R5任一→观望；黄旗Y1-Y5→降一档+最小注+仅长期限。
3. 用市场数据独立核查一个关键事实。
4. 历史验证胜率<45%下调置信度并在【修正说明】说明；>60%保持逻辑。

[置信度门槛(PA系统§0.1, G7, 动态)]
W*=1/(1+p)，p=平台净赔率(默认0.80→W*≈55.6%)；放行门槛=W*+5%安全垫≈60.6%。
置信度<门槛→观望；≥门槛→放行。(若平台赔率不同，按此公式重算，不要写死58%)

[期限层(PA系统§10, G6, 独立维度，定方向后单独决定)]
pullback→15/30M ; impulse高置信+放量→5/10M ; retrace_end→10M ; 区间反转→5/10M ; 挤压突破→10/15M ; 低置信/1H中性→15/30M。高ATR缩短一档。默认10M优先5M。

[只在三种情况观望(从严)]
①phase=exhaustion/reversal_risk与分析师顺势冲突 ②15M与30M主趋势矛盾且都无ADX≥22支撑 ③质疑师证实致命陷阱且无对冲证据。其余一律放行。趋势明确严禁因短期噪声观望。

[信号级别] 🟢A=趋势确认+相位顺势+回撤入场+无陷阱；🟡B=入场点一般或逼近关键位；🔴C=勉强放行(降期限或最小注)。

[字段一致性硬规则]
· 方向=看涨/看跌时【采纳方案】写顺势依据；观望写"观望，不入场"+①②③哪条
· 【市场状态】须与特征池marketState一致；趋势形成可标"趋势"；严禁把震荡标趋势

[输出格式，每行一个字段]
【方向】看涨/看跌/观望
【置信度】XX%
【信号级别】🟢A/🟡B/🔴C
【建议期限】5/10/15/30分钟或"无"
【市场状态】趋势/震荡/不明
【采纳方案】放行依据 或 "观望，不入场"+①②③哪条
【质疑师评估】质疑师是否指出被证实的致命陷阱(一句话)
【市场核查】独立核查的一个关键事实(一句话)
【信号摘要】方向+级别+置信度｜期限｜最强依据(≤30字)
【修正说明】原因或"无"
```

## 5. 决策流水线时序（PA系统 §2 漏斗 + §11 协作）

```
每根1m收盘 → 数据层产出 feature_pool(BINARY_OPTIONS_FEATURES_V3)
   │
   ▼
[历史学家] 读G0相位 → 同相位顺势先验
   │
   ▼
[分析师] G1环境路由 → G2方向引擎 → 给方向+入场点+期限初值
   │
   ▼
[质疑师] G3量价证伪 + G4反陷阱 → 五类致命核查(支持/拦截)
   │
   ▼
[裁判] G0相位闸门 → G5硬否决 → G7动态门槛 → G6期限层 → 最终 CALL/PUT/NO-TRADE
   │
   ▼
执行层下单 → 到期用真实K线验证 → 胜率回灌 + 元裁判(每15条) → 历史学家样本库
```

**仲裁优先级（冲突时谁说了算）**：
```
1. G0 相位闸门(suppress/pullback期限约束)  —— 最高，谁都不能推翻
2. 裁判 G5 红色否决                          —— 直接 NO-TRADE
3. 裁判 G7 动态门槛 + G6 期限                 —— 在1、2通过后定档
4. 分析师 G2 方向                            —— 提供候选
5. 历史学家先验                              —— 仅调整置信，不定方向
```

## 7. 验证与迭代（PA系统 §13.3 / §15.3）

1. **分桶胜率**：按「环境×相位」分桶（趋势-impulse / 趋势-pullback / 区间-边界 / 挤压-突破）。任一桶长期 <W\* 保本线→该桶禁用或拉长期限。
2. **错因归类**：每笔标注"方向错"(回G2查是否真顺势) vs "期限错"(回G6拉长期限)。
3. **元裁判校准**：≥30条已验证后，按相位统计 suppress 误杀率，反推 G0/G7 阈值。
4. **赔率自适应**：平台赔率 p 变化时，G7 门槛按 W\*=1/(1+p) 自动重算，不手改。
